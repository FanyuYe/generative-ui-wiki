The [AG-UI Dojo application](https://dojo.ag-ui.com/) provides an interactive environment to test and explore Microsoft Agent Framework agents that implement the AG-UI protocol. Dojo offers a visual interface to connect to your agents and interact with all 7 AG-UI features.

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#prerequisites)
## Prerequisites

Before you begin, ensure you have:

*   Python 3.10 or higher
*   [uv](https://docs.astral.sh/uv/) for dependency management
*   An OpenAI API key or Azure OpenAI endpoint
*   Node.js and pnpm (for running the Dojo frontend)

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#installation)
## Installation

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#1-clone-the-ag-ui-repository)
### 1. Clone the AG-UI Repository

First, clone the AG-UI repository which contains the Dojo application and Microsoft Agent Framework integration examples:

 Bash 

Copy

```bash
git clone https://github.com/ag-oss/ag-ui.git
cd ag-ui
```

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#2-navigate-to-examples-directory)
### 2. Navigate to Examples Directory

 Bash 

Copy

```bash
cd integrations/microsoft-agent-framework/python/examples
```

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#3-install-python-dependencies)
### 3. Install Python Dependencies

Use `uv` to install the required dependencies:

 Bash 

Copy

```bash
uv sync
```

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#4-configure-environment-variables)
### 4. Configure Environment Variables

Create a `.env` file from the provided template:

 Bash 

Copy

```bash
cp .env.example .env
```

Edit the `.env` file and add your API credentials:

 Python 

Copy

```python
# For OpenAI
OPENAI_API_KEY=your_api_key_here
OPENAI_CHAT_COMPLETION_MODEL="gpt-4.1"

# Or for Azure OpenAI
AZURE_OPENAI_ENDPOINT=your_endpoint_here
AZURE_OPENAI_API_KEY=your_api_key_here
AZURE_OPENAI_CHAT_COMPLETION_MODEL=your_deployment_here
```

Note

If using `DefaultAzureCredential`, in place for an `api_key` for authentication, make sure you're authenticated with Azure (e.g., via `az login`). For more information, see the [Azure Identity documentation](https://learn.microsoft.com/en-us/python/api/azure-identity/azure.identity.defaultazurecredential).

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#running-the-dojo-application)
## Running the Dojo Application

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#1-start-the-backend-server)
### 1. Start the Backend Server

In the examples directory, start the backend server with the example agents:

 Bash 

Copy

```bash
cd integrations/microsoft-agent-framework/python/examples
uv run dev
```

The server will start on `http://localhost:8888` by default.

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#2-start-the-dojo-frontend)
### 2. Start the Dojo Frontend

Open a new terminal window, navigate to the root of the AG-UI repository, and then to the Dojo application directory:

 Bash 

Copy

```bash
cd apps/dojo
pnpm install
pnpm dev
```

The Dojo frontend will be available at `http://localhost:3000`.

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#3-connect-to-your-agent)
### 3. Connect to Your Agent

1.   Open `http://localhost:3000` in your browser

2.   Configure the server URL to `http://localhost:8888`

3.   Select "Microsoft Agent Framework (Python)" from the dropdown

4.   Start exploring the example agents

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#available-example-agents)
## Available Example Agents

The integration examples demonstrate all 7 AG-UI features through different agent endpoints:

Expand table

| Endpoint | Feature | Description |
| --- | --- | --- |
| `/agentic_chat` | Feature 1: Agentic Chat | Basic conversational agent with tool calling |
| `/backend_tool_rendering` | Feature 2: Backend Tool Rendering | Agent with custom tool UI rendering |
| `/human_in_the_loop` | Feature 3: Human in the Loop | Agent with approval workflows |
| `/agentic_generative_ui` | Feature 4: Agentic Generative UI | Agent that breaks down tasks into steps with streaming updates |
| `/tool_based_generative_ui` | Feature 5: Tool-based Generative UI | Agent that generates custom UI components |
| `/shared_state` | Feature 6: Shared State | Agent with bidirectional state synchronization |
| `/predictive_state_updates` | Feature 7: Predictive State Updates | Agent with predictive state updates during tool execution |

## Testing Your Own Agents

To test your own agents with Dojo:

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#1-create-your-agent)
### 1. Create Your Agent

Create a new agent following the [Getting Started](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/getting-started) guide:

 Python 

Copy

```python
import os
from agent_framework import Agent
from agent_framework.openai import OpenAIChatCompletionClient

# Create your agent
chat_client = OpenAIChatCompletionClient(
    model=os.getenv("AZURE_OPENAI_CHAT_COMPLETION_MODEL"),
    azure_endpoint=os.getenv("AZURE_OPENAI_ENDPOINT"),
    api_version=os.getenv("AZURE_OPENAI_API_VERSION"),
    api_key=os.getenv("AZURE_OPENAI_API_KEY"),
)

agent = Agent(
    name="my_test_agent",
    client=chat_client,
    system_message="You are a helpful assistant.",
)
```

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#2-add-the-agent-to-your-server)
### 2. Add the Agent to Your Server

In your FastAPI application, register the agent endpoint:

 Python 

Copy

```python
from fastapi import FastAPI
from agent_framework_ag_ui import add_agent_framework_fastapi_endpoint
import uvicorn

app = FastAPI()

# Register your agent
add_agent_framework_fastapi_endpoint(
    app=app,
    path="/my_agent",
    agent=agent,
)

if __name__ == "__main__":
    uvicorn.run(app, host="127.0.0.1", port=8888)
```

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#3-test-in-dojo)
### 3. Test in Dojo

1.   Start your server
2.   Open Dojo at `http://localhost:3000`
3.   Set the server URL to `http://localhost:8888`
4.   Your agent will appear in the endpoint dropdown as "my_agent"
5.   Select it and start testing

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#project-structure)

## Troubleshooting

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#server-connection-issues)
### Server Connection Issues

If Dojo can't connect to your server:

*   Verify the server is running on the correct port (default: 8888)
*   Check that the server URL in Dojo matches your server address
*   Ensure no firewall is blocking the connection
*   Look for CORS errors in the browser console

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#agent-not-appearing)
### Agent Not Appearing

If your agent doesn't appear in the Dojo dropdown:

*   Verify the agent endpoint is registered correctly
*   Check server logs for any startup errors
*   Ensure the `add_agent_framework_fastapi_endpoint` call completed successfully

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#environment-variable-issues)
### Environment Variable Issues

If you see authentication errors:

*   Verify your `.env` file is in the correct directory
*   Check that all required environment variables are set
*   Ensure API keys and endpoints are valid
*   Restart the server after changing environment variables

[](https://learn.microsoft.com/en-us/agent-framework/integrations/ag-ui/testing-with-dojo#next-steps)
